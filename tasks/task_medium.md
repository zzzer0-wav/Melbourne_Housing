Підказка кодом (кореляція) - це окремий важливий інструмент, тож докладніше.

1. Кореляція ознак саме з ціллю. Скористайтеся `.select_dtypes("number")`, `.corr()`, далі стовпець `["Price"]`, `.drop("Price")`, `.sort_values(key=abs, ascending=False)`:

```python

```

2. Як читати числа:
   - **знак** (`+` чи `-`) - напрям зв'язку (зростає разом із ціною чи навпаки);
   - **модуль** (близькість до 1) - сила лінійного зв'язку; близько 0 - лінійного зв'язку майже немає;
   - грубі орієнтири: `|r|` понад 0.5 - помітний зв'язок, 0.3-0.5 - помірний, менше 0.2 - слабкий.

3. Пошук майже-дублікатів ознак. Тут потрібна кореляція **ознак між собою**, а не лише з ціллю. Побудуйте повну матрицю і знайдіть пари з дуже високим `|r|` (наприклад, понад 0.9) - їх немає сенсу тримати разом:

```python
cm = num.corr().abs()
# подивіться на матрицю cm і знайдіть підозріло високі пари,
# зокрема Rooms - Bedroom2
```

   Для наочності можна вивести `num.corr().round(2)` або побудувати теплову карту через `plt.imshow(num.corr())` з підписами осей.

4. Важливе застереження: кореляція ловить **лише лінійні** зв'язки. Зв'язок може бути сильним, але нелінійним (тоді `r` близький до 0) - тому графіки з пункту вище не менш важливі, ніж числа. І як завжди: кореляція не означає причинність.
